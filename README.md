# SURE Trust FSD Group

This repository is a shared learning workspace for SURE Trust Full Stack Development and DSA Java learners. It brings together task submissions, live-session examples, learning notes, and practice projects in one place so mentees can learn by reading, building, comparing, and improving real code.

## Repository Overview

The repository is organized around cohort-wise tasks, reusable learning material, and mentor-led demonstrations.

| Folder | Purpose |
| --- | --- |
| `G30 FSD - Tasks/` | Task submissions from the G30 Full Stack Development cohort. |
| `G31 FSD - Tasks/` | Task submissions from the G31 Full Stack Development cohort. |
| `G4 DSA Java - Tasks/` | DSA Java and frontend-related task submissions from the G4 cohort. |
| `Learning Resources/` | Notes and guides for HTML5, CSS3, JavaScript, Angular, Python, and SQL. |
| `Live Sessions - Demonstration/` | Code examples used during live learning sessions. |
| `Mentee Contributions/` | Additional resources and examples contributed by mentees. |

## How to Use This Repository

1. Open the folder for your cohort, such as `G30 FSD - Tasks/`, `G31 FSD - Tasks/`, or `G4 DSA Java - Tasks/`.
2. Choose the task folder you want to study or submit to.
3. Open individual mentee folders to review implementations.
4. For HTML/CSS/JavaScript projects, open the relevant `index.html` file in a browser.
5. For projects that use package files such as `package.json`, install dependencies before running the project.

Example for a Node-based demo:

```bash
npm install
npm run dev
```

If a task uses an API key, check for files such as `config.example.js` or instructions in the task folder before running it.

## Contribution Guidelines

- Keep your work inside the correct cohort and task folder.
- Use a clear folder name, preferably in this format: `Your Name - Cohort Name`.
- Include all required files for your submission, such as `index.html`, `style.css`, `script.js`, screenshots, or notes.
- Do not commit private API keys, passwords, tokens, or personal credentials.
- Add a short `README.md` inside your submission folder when setup steps or API configuration are needed.
- Keep filenames simple and meaningful.
- Test your project locally before submitting.

## Recommended Submission Structure

```text
Task Name/
`-- Your Name - Cohort/
    |-- index.html
    |-- style.css
    |-- script.js
    `-- README.md
```

## Learning Approach

Mentees are encouraged to:

- Compare different submissions for the same task.
- Read live-session demos before starting assignments.
- Use learning resources as quick references.
- Practice Git regularly through small, clean commits.
- Improve existing work by refactoring, documenting, and testing.
